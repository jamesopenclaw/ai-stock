# 微信小程序测试验收标准

## 测试范围

| 优先级 | 页面/功能 | 后端 API |
|--------|-----------|----------|
| P0 | 首页（今日摘要） | /api/v1/market/env, /api/v1/sector/scan |
| P0 | 市场环境 | /api/v1/market/env |
| P0 | 板块扫描 | /api/v1/sector/scan |
| P0 | 三池 | /api/v1/stock/pools |
| P1 | 买点详情 | /api/v1/decision/buy-point |
| P1 | 卖点详情 | /api/v1/decision/sell-point |
| P2 | 账户持仓 | /api/v1/account/profile, /api/v1/account/positions |

---

## P0: 首页（今日摘要）

### 功能验收
- [ ] 能展示当前交易日日期
- [ ] 能展示市场环境标签（进攻/中性/防守）
- [ ] 能展示风险等级（低/中/高）
- [ ] 能展示主线板块名称（至少1个）
- [ ] 能展示可参与池股票数量
- [ ] 能展示需要处理的持仓数量

### 交互验收
- [ ] 页面加载时显示 loading 状态
- [ ] 数据加载完成后自动更新
- [ ] 下拉刷新能重新加载数据
- [ ] 展示简洁的今日操作建议

### 数据展示验收
- [ ] 环境标签用正确颜色区分（绿=进攻，黄=中性，红=防守）
- [ ] 数字展示格式正确（涨跌幅用 %，金额用万/亿）
- [ ] 无数据时显示占位提示

---

## P0: 市场环境

### 功能验收
- [ ] 能获取当前市场环境判定
- [ ] 正确展示 market_env_tag（进攻/中性/防守）
- [ ] 正确展示 breakout_allowed（是否允许突破）
- [ ] 正确展示 risk_level（低/中/高）
- [ ] 正确展示市场评论

### 交互验收
- [ ] 支持切换查看不同交易日数据
- [ ] 页面刷新能重新获取数据
- [ ] API 请求失败时显示错误提示

### 数据展示验收
- [ ] 环境标签视觉化展示（颜色/图标）
- [ ] breakout_allowed 展示为"允许"/"不允许"或对应图标
- [ ] risk_level 用颜色区分（绿=低，黄=中，红=高）
- [ ] market_comment 正确换行显示

---

## P0: 板块扫描

### 功能验收
- [ ] 能获取板块扫描结果
- [ ] 正确展示主线板块列表
- [ ] 正确展示次主线板块列表
- [ ] 正确展示跟风板块列表
- [ ] 正确展示杂毛板块列表
- [ ] 展示板块涨跌幅

### 交互验收
- [ ] 板块列表支持滚动加载
- [ ] 点击板块能查看详情（可选）
- [ ] 支持按涨跌幅排序
- [ ] 页面刷新能重新获取数据

### 数据展示验收
- [ ] 板块按涨幅降序排列
- [ ] 涨跌幅用正确颜色区分（红涨绿跌）
- [ ] 主线板块有明确标识
- [ ] 板块数量统计正确

## P0: 三池

### 功能验收
- [ ] 能获取三池分类结果
- [ ] 正确展示市场最强观察池
- [ ] 正确展示账户可参与池
- [ ] 正确展示持仓处理池

### 交互验收
- [ ] 三个池之间支持切换
- [ ] 每个池支持滚动查看
- [ ] 点击股票能查看详情（可选）

### 数据展示验收
- [ ] 每个池的股票数量正确
- [ ] 股票信息展示完整（代码、名称、涨跌幅）
- [ ] 空池时显示占位提示
- [ ] 池标签区分清晰

## P1: 买点详情

### 功能验收
- [ ] 能获取买点分析结果
- [ ] 正确展示可买候选列表
- [ ] 正确展示观察候选列表

- [ ] 每个买点的确认条件正确展示

## P1: 卖点详情

### 功能验收
- [ ] 能获取卖点分析结果
- [ ] 正确展示持仓列表
- [ ] 每个持仓有明确的操作建议（持有/减仓/卖出/观察）
- [ ] 卖点类型分类正确（止损/止盈/减仓/观察）

### 数据展示验收
- [ ] 持仓股票信息完整（代码、名称、成本价、现价、盈亏）
- [ ] 盈亏用颜色区分（红=盈利，绿=亏损）
- [ ] T+1 锁定的持仓有明确标识
- [ ] 卖点触发条件正确展示

### 交互验收
- [ ] 点击持仓能查看详情
- [ ] 不同操作建议有视觉区分

---

## P2: 账户持仓管理

### 功能验收
- [ ] 能获取账户概况（总资产、可用资金、仓位、持仓数）
- [ ] 能获取持仓明细列表
- [ ] 正确展示 T+1 锁定数量

### 数据展示验收
- [ ] 总资产、可用资金格式正确
- [ ] 仓位用百分比展示
- [ ] 持仓数量统计正确
- [ ] 持仓明细完整（代码、名称、数量、成本、市值、盈亏）

### 交互验收
- [ ] 持仓列表支持滚动
- [ ] 点击持仓能查看详情

## API 通用验收

### 网络验收
- [ ] 所有 API 请求有 loading 状态
- [ ] 网络错误时显示友好提示
- [ ] 接口超时处理合理（< 10s）

### 数据验收
- [ ] API 返回的 JSON 结构正确
- [ ] 必要字段缺失时有容错处理
- [ ] 数据类型展示格式正确
