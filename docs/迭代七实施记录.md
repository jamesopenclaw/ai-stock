## 1. 迭代目标

迭代七的目标是把系统从“能用”推进到“更稳、更容易验收”，重点补齐：

- 登录与首个请求的稳定性缓解
- 关键链路的耗时可观测性
- 前后端可重复执行的回归约束

---

## 2. 本轮完成项

### 2.1 登录冷启动缓解

- 新增轻量预热端点 `GET /api/v1/ping`
- 登录页挂载时静默预热 API
- 将“用户点击登录之前的冷连接成本”提前吃掉，缓解首个登录慢的问题

对应文件：

- [app/main.py](../app/main.py)
- [frontend/src/api/index.js](../frontend/src/api/index.js)
- [frontend/src/views/Login.vue](../frontend/src/views/Login.vue)

### 2.2 认证链路可观测性

- 为认证关键链路增加耗时日志：
  - `auth.authenticate_user`
  - `auth.create_refresh_session`
  - `auth.rotate_refresh_session`
  - `auth.login`
  - `auth.refresh`
  - `auth.logout`
  - 慢的 `get_current_user`
  - 慢的 `get_current_account`
- 日志可区分数据库查询、密码校验、session 写入和整体接口耗时

对应文件：

- [app/core/security.py](../app/core/security.py)

## 5. 遗留建议

- 当前对“首个登录慢”的处理仍以预热缓解为主，不算根因级修复
- 若后续要继续深挖稳定性，应优先看：
  - 容器冷连接
  - 数据库连接池首个请求
  - 首个健康检查与首个登录请求的共因
- 可继续补充：
  - `/auth/refresh` 集成回归
  - 管理员后台接口集成回归
  - 更完整的浏览器级 smoke 流程
