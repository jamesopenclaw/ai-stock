# 迭代六权限与回归清单

**文档状态**：执行约束  
**创建日期**：2026-03-29  
**关联迭代**：[多账户支持迭代计划](./多账户支持迭代计划.md)

---

## 1. 目标

本清单用于固定迭代六完成后的两类结果：

- 角色权限边界
- 关键页面回归路径

后续若要调整菜单、系统接口、缓存或复盘页信息架构，应先对照本清单；若结果与本清单冲突，应先更新文档再改代码。

---

## 2. 角色权限矩阵

### 2.1 普通用户可见页面

- Dashboard
- 市场环境
- 板块扫描
- 三池分类
- 买点分析
- 卖点分析
- 账户管理
- 复盘统计
- LLM调用记录

说明：

- `LLM调用记录` 当前允许普通用户访问，但只能看到自己当前账户的数据。

### 2.2 普通用户不可见页面

- 系统设置
- 任务调度
- 用户管理
- 交易账户

### 2.3 管理员可见页面

- 普通用户全部页面
- 系统设置
- 任务调度
- 用户管理
- 交易账户

### 2.4 管理员专属接口

- `GET /api/v1/admin/users`
- `POST /api/v1/admin/users`
- `PUT /api/v1/admin/users/{id}`
- `GET /api/v1/admin/accounts`
- `POST /api/v1/admin/accounts`
- `PUT /api/v1/admin/accounts/{id}`
- `POST /api/v1/admin/accounts/{id}/bind-user`
- `POST /api/v1/task/trigger`
- `GET /api/v1/task/status`

### 2.5 账户内可用接口

- `/api/v1/account/*`
- `/api/v1/decision/*`
- `/api/v1/stock/pools`
- `/api/v1/system/llm/logs`

约束：

- 上述接口必须受当前登录账户上下文约束
- 普通用户不能借助 `X-Account-Id` 越权查看其他账户

## 3. 前端可见性约束

### 3.1 菜单约束

以下菜单仅管理员可见：

- `系统设置`
- `任务调度`
- `用户管理`
- `交易账户`

### 3.2 路由约束

普通用户手动输入以下 URL 也必须被拦截：

- `/system`
- `/tasks`
- `/admin/users`
- `/admin/accounts`

---

## 4. 复盘页信息架构约束

### 4.1 类型文案

复盘统计中的 `类型` 必须显式说明来源，不允许只显示模糊标签。

当前标准文案：

- `买点-可买`
- `买点-观察`
- `三池-可参与池`
- `三池-观察池`

### 4.2 分层文案

复盘统计中的 `分层` 必须同时展示：

- 原始结构标签
- 一句用户可理解的结构解释

示例：

- `强势确认`：重点看放量确认和站稳，不适合无脑追高
- `趋势回踩`：重点看回踩承接和再次转强，适合等舒服位置

### 4.3 来源跳转

从复盘统计跳转到源页面时：

- 跳到买点页应自动打开对应 `可买` 或 `观察` tab
- 跳到三池页应自动打开对应 `可参与池` 或 `观察池` tab
- 页面需要保留 `review_source` 与 `review_bucket` 筛选上下文

## 5. 缓存与性能约束

### 5.1 必须按账户隔离的前端缓存

- Dashboard 首页缓存
- 复盘统计缓存
- 三池缓存
- 买点页缓存

### 5.2 已接入短 TTL 缓存与并发去重的共享接口

- `GET /api/v1/market/env`
- `GET /api/v1/decision/review-stats`
- `GET /api/v1/stock/pools`

要求：

- 页面普通进入优先复用缓存
- 用户主动点击“刷新”时必须绕过缓存

## 6. 回归清单

说明：

- 已自动化项会标注“已自动化”
- 未标注项默认为仍需人工回归
- 当前前端 smoke 共 8 条

### 6.1 权限回归

普通用户登录：

- 看不到 `系统设置`  已自动化
- 看不到 `任务调度`  已自动化
- 看不到 `用户管理`  已自动化
- 看不到 `交易账户`  已自动化
- 直链访问 `/system` 被拦截
- 直链访问 `/tasks` 被拦截
- 直链访问 `/admin/users` 被拦截
- 直链访问 `/admin/accounts` 被拦截

管理员登录：

- 能看到全部菜单
- 能进入用户管理
- 能进入交易账户
- 能触发任务调度

### 6.2 多账户回归

- 管理员切换账户后，Dashboard 不显示上一个账户的数据  已自动化（账户切换本地状态）
- 账户页显示当前账户的持仓和资产
- 复盘统计与当前账户一致
- LLM 调用记录不串账户

### 6.3 复盘链路回归

- 复盘页能区分三池来源和买点来源
- 复盘页能解释分层含义
- 点击“去源页面”能跳到正确页面  已自动化
- 跳转后自动定位到正确 tab
- 清除筛选后页面恢复正常视图

### 6.4 性能回归

- Dashboard 切页返回时，不重复请求 `market/env` 和 `review-stats`
- 复盘页首次加载后短时间刷新，不重复拉同一份统计
- 三池页从复盘页跳入后，短时间内不重复重拉同一份三池结果
- 用户点击“刷新”按钮时，仍然可以拿到最新数据

## 7. 建议测试顺序

1. 普通用户登录，验证菜单和路由拦截
2. 管理员登录，验证菜单和管理员页
3. 管理员切换账户，验证 Dashboard / Account / Review / LLM 日志
4. 从复盘页跳转到三池和买点，验证筛选和默认 tab
5. 在 Dashboard、Review、Pools 之间往返切换，观察是否重复请求共享接口
