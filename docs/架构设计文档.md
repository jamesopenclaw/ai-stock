## 1. 技术栈选型

### 1.1 技术选型原则

- **轻量优先**：个人投资者系统，无需支撑高并发，优先选择开发效率高、维护成本低的方案
- **数据为核心**：系统是数据密集型，核心在于数据获取、清洗、计算逻辑
- **可演进**：预留扩展空间，避免早期过度设计

### 1.2 技术栈明细

| 层次 | 技术选型 | 选型理由 |
|------|----------|----------|
| **后端框架** | FastAPI (Python 3.11+) | 高性能异步框架，天然支持数据处理任务，API设计规范，自动生成OpenAPI文档 |
| **数据处理** | Pandas + NumPy | A股数据分析事实标准，Series/DataFrame操作便捷 |
| **前端** | Vue 3 + Element Plus | 轻量级管理后台，组件丰富，适合内部工具 |
| **数据库** | PostgreSQL 15 + Redis | PG存储结构化数据+历史行情，Redis缓存热点数据+任务队列 |
| **数据源** | A股行情API (tushare/akshare) | 免费/低成本获取日频/分钟级行情数据 |
| **任务调度** | APScheduler | 内置任务调度，支持定时执行每日收盘分析 |
| **部署** | Docker + Docker Compose | 简化部署，一键启动全部服务 |

## 2. 系统架构设计

### 2.1 核心模块划分

系统划分为 **6 大核心模块**，每个模块独立职责，通过事件/数据流协作：

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **市场环境模块** | 判断当日市场整体状态 | 指数涨跌、涨停跌停数、成交额、风险偏好 | 环境标签(进攻/中性/防守)、风险等级、 breakout_allowed |
| **板块扫描模块** | 识别主线板块、评估持续性 | 板块行情数据、催化信息 | 板块分类标签、交易性标签、证伪线索 |
| **个股筛选模块** | 从有效板块中筛选可交易标的 | 个股行情、所属板块、账户约束 | 个股强弱标签、三池分类、证伪条件 |
| **买点分析模块** | 判定具体买点的触发/失效条件 | 市场环境+板块+个股+账户 | 买点类型、触发条件、确认条件、失效条件 |
| **卖点分析模块** | 持仓卖点判定与优先级 | 持仓明细、市场环境、个股走势 | 卖出/减仓/持有建议、失效信号 |
| **账户适配模块** | 账户约束下的可执行性判断 | 账户资产、仓位、持仓、新开仓限制 | 可执行标记、仓位压力、新开仓许可 |

### 2.3 三池分离机制

**核心设计原则**：每日选股结果必须拆分为三个独立池，禁止混写。

| 池子 | 定义 | 筛选条件 | 用途 |
|------|------|----------|------|
| **市场最强观察池** | 仅从市场角度评估最强标的 | 市场强+板块强+个股强，不考虑账户 | 观察方向，理解市场 |
| **账户可参与池** | 叠加账户约束后的可执行候选 | 观察池 - 账户不适合的标的 | 实际交易建议 |
| **持仓处理池** | 已有持仓的处理建议 | 全部持仓逐一分析 | 卖出/减仓/持有决策 |

### 2.4 数据流转设计

```
数据输入层
    │
    ├── 行情数据: tushare/akshare → DataLoader → Redis缓存
    │
    ├── 账户数据: 轻舟API → AccountService → 账户状态快照
    │
    └── 用户配置: 管理后台 → 配置入库

处理层 (Pipeline)
    │
    ├── Step1: MarketEnvService.process()
    │   └── 输出: market_env_tag, breakout_allowed, risk_level
    │
    ├── Step2: SectorScanService.process()
    │   └── 输出: List[SectorAnalysis]
    │
    ├── Step3: StockFilterService.process()
    │   └── 输出: List[StockAnalysis] + pool_tag
    │
    ├── Step4: BuyPointService.process()
    │   └── 输出: List[BuyPoint]
    │
    ├── Step5: SellPointService.process()
    │   └── 输出: List[SellPoint]
    │
    └── Step6: AccountAdapterService.process()
        └── 输出: final_action_tag, priority_action

输出层
    │
    └── DecisionOutput (结构化 JSON)
```

### 2.5 核心决策逻辑

#### 2.5.1 市场环境判定

```python
# 伪代码: 市场环境判定逻辑
def judge_market_env(index_data, sentiment_data) -> MarketEnv:
    # 防守条件
    if index_data.all_down and sentiment_data.bad:
        return MarketEnv(tag="防守", breakout_allowed=False, risk_level="高")
    
    # 进攻条件
    if index_data.resilient and sentiment_data.good and sector_data.mainline_clear:
        return MarketEnv(tag="进攻", breakout_allowed=True, risk_level="低")
    
    # 中性
    return MarketEnv(tag="中性", breakout_allowed=False, risk_level="中")
```

#### 2.5.2 买点有效性判定

```python
# 伪代码: 买点过滤器
def filter_buy_point(buy_point, market_env, account_state) -> BuySignal:
    # 环境过滤
    if market_env.tag == "防守" and buy_point.type == "突破":
        return BuySignal(tag="不买", reason="弱市不做突破")
    
    # 账户过滤
    if account_state.position_pressure == "高":
        return BuySignal(tag="不买", reason="仓位压力过大")
    
    # 失效条件检查
    if buy_point.invalid_cond_now_met:
        return BuySignal(tag="不买", reason="失效条件已触发")
    
    return BuySignal(tag="可买", ...)
```

## 3. API 接口设计

### 3.1 API 设计规范

- **Base URL**: `/api/v1`
- **认证方式**: API Key (后续可扩展 JWT)
- **响应格式**: JSON
- **错误码**: 遵循 HTTP 标准 + 业务错误码

### 3.2 核心 API 清单

|------|------|------|
| GET | `/api/v1/market/env` | 获取当日市场环境判定 |

## 6. 核心算法要点

### 6.1 市场环境评分算法

```python
def calculate_market_score(index_data, sentiment_data) -> dict:
    score = 0
    
    # 指数维度 (权重 40%)
    index_score = (index_data.sh_change * 0.4 + 
                   index_data.sz_change * 0.3 + 
                   index_data.cyb_change * 0.3)
    score += index_score * 40
    
    # 情绪维度 (权重 30%)
    sentiment_score = (
        (sentiment_data.up_down_ratio - 0.5) * 30 +  # 涨跌比
        (1 - sentiment_data.broken_board_rate) * 20 + # 炸板率
        (1 - sentiment_data.limit_down_count / 100) * 10  # 跌停数量
    )
    score += sentiment_score * 0.3
    
    # 成交额维度 (权重 20%)
    if sentiment_data.turnover > 800000000000:  # 8000亿
        score += 20
    elif sentiment_data.turnover > 600000000000:
        score += 15
    elif sentiment_data.turnover > 400000000000:
        score += 10
    
    # 涨停维度 (权重 10%)
    if sentiment_data.limit_up_count > 100:
        score += 10
    elif sentiment_data.limit_up_count > 50:
        score += 7
    else:
        score += 3
    
    return {
        "score": score,
        "tag": "进攻" if score >= 70 else "中性" if score >= 40 else "防守",
        "risk_level": "低" if score >= 70 else "中" if score >= 40 else "高"
    }
```

### 6.2 板块强度评分算法

```python
def calculate_sector_strength(sector_data, history_data) -> dict:
    # 当日强度 (40%)
    daily_score = sector_data.change_pct * 40 / 5  # 5%涨幅得40分
    
    # 连续性 (30%)
    continuity_score = min(history_data.continuity_days * 5, 30)
    
    # 成交额 (20%)
    turnover_score = min(sector_data.turnover / 1000000000, 20)
    
    # 涨停家数 (10%)
    limit_up_score = min(sector_data.limit_up_stock_count * 2, 10)
    
    total_score = daily_score + continuity_score + turnover_score + limit_up_score
    
    tradeability = "可交易" if total_score >= 70 else "谨慎" if total_score >= 50 else "不建议"
    
    return {
        "score": total_score,
        "mainline_tag": "主线" if total_score >= 80 else 
                        "次主线" if total_score >= 60 else 
                        "跟风" if total_score >= 40 else "杂毛",
        "tradeability": tradeability
    }
```

### 6.3 个股筛选评分算法

```python
def calculate_stock_score(stock_data, sector_data, account_data) -> StockScore:
    base_score = 0
    
    # 个股维度 (50%)
    if stock_data.change_pct > 5:
        base_score += 20
    elif stock_data.change_pct > 3:
        base_score += 15
    elif stock_data.change_pct > 0:
        base_score += 10
    
    if stock_data.turnover_rate > 10:
        base_score += 15
    elif stock_data.turnover_rate > 5:
        base_score += 10
    
    if stock_data.vol_ratio > 2:
        base_score += 10
    elif stock_data.vol_ratio > 1.5:
        base_score += 5
    
    # 板块维度 (30%)
    if sector_data.mainline_tag == "主线":
        base_score += 30
    elif sector_data.mainline_tag == "次主线":
        base_score += 20
    elif sector_data.mainline_tag == "跟风":
        base_score += 10
    
    # 账户适配维度 (20%)
    if account_data.available_cash > 100000:
        base_score += 10
    
    if account_data.total_position_ratio < 0.7:
        base_score += 10
    
    return StockScore(
        score=base_score,
        strength_tag="强" if base_score >= 70 else "中" if base_score >= 40 else "弱",
        pool_tag="市场最强观察池" if base_score >= 70 else 
                 "账户可参与池" if base_score >= 50 else 
                 "不入池"
    )
```
