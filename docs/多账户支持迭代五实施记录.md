# 多账户支持迭代五实施记录

## 本轮已完成

- `review_snapshots` 增加 `account_id`，并按共享快照与账户快照分 scope 读写
- `stock_pool_snapshots` 增加 `account_id`，三池页快照按账户隔离
- `llm_cache_entries` 增加 `account_id`，账户相关 LLM 缓存不再混用
- `llm_call_logs` 增加 `account_id`，普通用户仅能查看当前账户日志
- `/decision/review-stats`、`/stock/pools`、`/decision/*` 读写快照时透传当前账户
- `get_current_account` 支持通过 `X-Account-Id` 显式切换目标账户
- Web 顶栏增加管理员账户切换器
- 管理员账户页增加“查看”动作，可直接切换到目标账户
- 小程序请求层补齐 `X-Account-Id` 注入、401 清理登录态、启动时 `me` 校验

## 本轮修复

- 修复 `create_trading_account()` 在同事务中先插入 `account_settings` 导致的外键失败
- 修复快照安全包装器把 `account_id=None` 透传给旧测试桩导致的兼容问题
- 修复买点分析、完整分析、定时任务在快照写入失败时直接中断主流程的问题
- 新增迁移 `0007_backfill_legacy_scope`，把单账户时代遗留的空 `account_id` 历史快照、三池快照、LLM 缓存与日志回填到默认账户，避免迁移后历史统计被账户过滤规则隐藏
