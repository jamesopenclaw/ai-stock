# 轻舟版交易系统

> 面向 A 股短线交易的高质量半自动决策辅助系统

## 是什么？

这是一个面向 **A 股短线交易**的决策辅助系统，用 Tushare 做市场数据分析，用手工维护的轻舟账户做账户适配，输出市场环境、选股结果、买点条件、卖点建议和执行摘要。

它**不是**：
- ❌ 荐股器
- ❌ 涨幅榜搬运工
- ❌ 自动交易机器人

它**解决的是**：
- ✅ 选股和账户脱节：市场强，不代表轻舟账户现在就适合做
- ✅ 买点定义模糊：不是"感觉能涨"就买，而是要有触发、确认、失效条件
- ✅ 卖点靠拖：把止损、止盈、减仓、观察分开
- ✅ 弱市乱出手：先看市场环境开关，弱市先防守

## 核心输出

每次分析输出：

| 输出 | 说明 |
|------|------|
| **市场最强观察池** | 仅从市场角度评估最强标的，不考虑账户 |
| **账户可参与池** | 叠加账户约束后的可执行候选 |
| **持仓处理池** | 已有持仓的卖出/减仓/持有建议 |
| **买点条件** | 触发条件、确认条件、失效条件 |
| **卖点建议** | 止损、止盈、减仓、观察 |
| **执行摘要** | 今天该不该出手、优先看谁、先处理谁 |

## 快速开始

### 环境要求

- Python 3.11+
- PostgreSQL 15+
- Redis 7+
- Docker & Docker Compose

### 安装

```bash
# 1. 克隆仓库
git clone https://github.com/jamesopenclaw/ai-stock.git
cd ai-stock

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入 TUSHARE_TOKEN

# 3. 启动服务
docker-compose up -d
```

### 配置说明

| 变量 | 说明 |
|------|------|
| `TUSHARE_TOKEN` | Tushare API Token |
| `DATABASE_URL` | PostgreSQL 连接字符串 |
| `REDIS_URL` | Redis 连接字符串 |

## 核心设计原则

### 1. 账户适配优先

系统在给出任何交易建议前，必须同时回答：
- 市场上谁最强？
- 当前账户能不能参与？
- 参与后会不会破坏仓位结构？

### 2. 三池分离

每日选股结果必须拆分为三类：
- **市场最强观察池** - 不考虑账户
- **账户可参与池** - 叠加账户约束
- **持仓处理池** - 已有持仓处理

### 3. 市场环境优先

系统必须先判断市场环境，再进行个股分析。市场环境作为全局策略开关，决定是否允许主动出手。

### 4. 不机械交易

所有买点必须结合：环境 + 板块 + 个股结构 + 量价关系 + 承接情况 + 账户状态

### 5. 风险先于机会

弱市先防守，强市再进攻。持仓风险与流动性风险优先级高于新开机会。
